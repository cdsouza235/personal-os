# Roadmap

## Phase -1: Repository Scaffold

Status: current

Scope:

- Create initial repository structure.
- Add placeholder documentation.
- Establish safety policy.
- Add inert placeholders for future modules.

Exit criteria:

- Scaffold and documentation are committed.
- No live-system inventory has been performed.
- No production state has been inspected or mutated.

## Phase 0: Read-Only Inventory

Scope:

- Inventory local project files and approved runtime surfaces.
- Produce an evidence-backed map of current systems.
- Keep all interactions read-only.

Gate:

- Requires explicit approval before starting.

## Phase 1: Local Model and Schema

Scope:

- Define local domain models.
- Draft SQLite schema for structured runtime state.
- Create tests around pure, non-integrated behavior.
